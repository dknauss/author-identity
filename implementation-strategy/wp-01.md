WORK PACKAGE 01: Plugin scaffold and adapter interface

Goal:
- Establish the plugin skeleton, adapter interface, auto-detection logic, and public API. Everything else builds on this.

Primary targets:
- `byline-feed.php`
- `inc/namespace.php`
- `inc/interface-adapter.php`
- `inc/class-adapter-core.php`
- `inc/class-adapter-cap.php`
- `inc/class-adapter-ppa.php`
- `composer.json`
- `tests/phpunit/test-adapter-core.php`
- `tests/phpunit/test-adapter-cap.php`
- `tests/phpunit/test-adapter-ppa.php`

Problem statement:
- No WordPress plugin currently provides a normalized, plugin-agnostic interface for resolving multi-author data. Each multi-author plugin has its own API surface and object shapes. A unified adapter layer is the prerequisite for every output channel.

Planned changes:

1. Create plugin bootstrap (`byline-feed.php`):
   - Plugin header with wp.org metadata.
   - On `plugins_loaded`, run adapter detection in priority order.
   - Store detected adapter instance in a singleton or static accessor.
   - Conditionally load output modules (feed, schema, fediverse, rights) based on which are enabled.

2. Define adapter interface (`inc/interface-adapter.php`):
   ```php
   namespace Byline_Feed;

   interface Adapter {
       /**
        * Returns ordered array of normalized author objects for the given post.
        * @param \WP_Post $post
        * @return object[]
        */
       public function get_authors( \WP_Post $post ) : array;
   }
   ```

3. Implement core WordPress fallback adapter (`inc/class-adapter-core.php`):
   - Reads `$post->post_author`.
   - Returns single-element array with normalized object from `get_userdata()`.
   - Populates `role` via capability check (`edit_others_posts` → `staff`, else `contributor`).
   - Populates `fediverse` from user meta key `byline_feed_fediverse` if present.
   - Populates `ai_consent` from user meta key `byline_feed_ai_consent` if present.

4. Implement Co-Authors Plus adapter (`inc/class-adapter-cap.php`):
   - Calls `get_coauthors( $post->ID )`.
   - Maps `$coauthor->type === 'guest-author'` to `is_guest = true`, `role = 'guest'`.
   - Maps WP user capabilities to Byline roles for non-guest authors.
   - Populates identity fields from coauthor object properties.

5. Implement PublishPress Authors adapter (`inc/class-adapter-ppa.php`):
   - Detects PPA via `function_exists( 'publishpress_authors_get_post_authors' )` or `MultipleAuthors` class.
   - Maps PPA author objects (which carry `term_id`, `user_id`, `is_guest`, `display_name`).
   - Resolves profile data from term meta for guest authors, from user meta for linked users.

6. Register public API function (`inc/namespace.php`):
   ```php
   function byline_feed_get_authors( \WP_Post $post ) : array {
       $adapter = byline_feed_get_adapter();
       $authors = $adapter->get_authors( $post );
       return apply_filters( 'byline_feed_authors', $authors, $post );
   }
   ```

7. Register role mapping utility with filter:
   ```php
   function byline_feed_map_role( \WP_User $user, \WP_Post $post = null ) : string {
       if ( user_can( $user, 'edit_others_posts' ) ) {
           $role = 'staff';
       } elseif ( user_can( $user, 'edit_published_posts' ) ) {
           $role = 'contributor';
       } else {
           $role = 'contributor';
       }
       return apply_filters( 'byline_feed_role', $role, $user, $post );
   }
   ```

8. Write PHPUnit tests:
   - `test-adapter-core.php`: single author resolution, role mapping, missing user handling.
   - `test-adapter-cap.php`: mock `get_coauthors()`, verify normalized shape for mixed user/guest arrays.
   - `test-adapter-ppa.php`: mock PPA API, verify normalized shape including term meta resolution.
   - All tests verify: correct field types, ordering preserved, empty optional fields default correctly.

Validation:
- Plugin activates without errors on PHP 7.4 / WP 6.0.
- `byline_feed_get_authors( $post )` returns correct normalized arrays for:
  - Core WordPress (single author).
  - Co-Authors Plus (multiple authors, mixed user/guest).
  - PublishPress Authors (multiple authors, mixed user/guest).
  - Posts with no author (returns empty array).
- All adapters produce objects conforming to the normalized contract.
- PHPUnit suite green.
- PHPCS clean against WordPress coding standards.

Notes:
- Molongui and HM Authorship adapters are deferred to a later work package. The interface and detection logic should accommodate them but the adapter classes are not part of MVP.
- The adapter detection function and public API are the contract that every subsequent work package depends on. Get the interface right here.
