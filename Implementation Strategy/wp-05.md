WORK PACKAGE 05: JSON-LD schema output

Goal:
- Enrich structured author identity in schema.org output across three modes:
  (A) Yoast SEO active → hook Yoast's schema graph filters to add multi-author arrays,
      fediverse handles, profiles, role, perspective, and ai_consent signals directly into
      the graph that NLWeb's schemamap endpoint queries.
  (B) Rank Math active → extend via Rank Math schema filters, equivalent enrichment.
  (C) No SEO plugin → output own standalone Article + Person JSON-LD block.

Primary targets:
- `inc/schema.php`                         — shared builder functions and mode dispatch
- `inc/schema-yoast.php`                   — Yoast integration (filters + graph pieces)
- `inc/schema-rankmath.php`                — Rank Math integration
- `inc/schema-standalone.php`              — fallback standalone output
- `tests/phpunit/test-schema.php`          — full test coverage across all three modes

Depends on:
- WP-01 (adapter interface and normalized author data)
- WP-03 (perspective meta field — needed for Article additionalProperty enrichment)
- WP-06 (ai_consent per-author/per-post signals — needed for consent graph piece)
  Note: WP-03 and WP-06 enrichments are progressive — schema output functions correctly
  without them; they add additional fields when available.

Problem statement:
- Google's E-E-A-T framework rewards structured author identity, and AI agent discovery
  now depends on it too. In March 2026, Yoast launched Schema Aggregation (the "schemamap")
  in collaboration with Microsoft's NLWeb project — a single endpoint that exposes a
  WordPress site's entire schema graph for AI agent consumption via MCP. NLWeb explicitly
  lists RSS and Schema.org as its semantic foundation.
- Yoast's schemamap deduplicates and connects site-wide entities well, but has two gaps
  relevant to byline-feed: (1) the Article node's `author` field references a single Person
  even on multi-author posts; (2) the Person nodes carry no byline-specific signals —
  no contributor role, no editorial perspective, no fediverse identity from live plugin
  data, no AI consent status.
- The previous WP-05 design responded to Yoast detection by disabling our own schema
  output. That was the wrong default: it ceded the NLWeb schemamap endpoint entirely.
  The correct posture is the opposite — when Yoast is active, we enrich its graph directly,
  so our data reaches AI agents through the endpoint they are already querying.
- On sites without Yoast or Rank Math, standalone output remains the right path.

## Mode A: Yoast SEO integration (primary path)

Detection:

```php
function byline_feed_schema_yoast_active(): bool {
    return defined( 'WPSEO_VERSION' )
        && class_exists( 'Yoast\WP\SEO\Generators\Schema\Abstract_Schema_Piece' );
}
```

### A1 — Enrich Article node: replace single-author ref with multi-author array

Hook: `wpseo_schema_article` (filters the assembled Article data array before output).

```php
add_filter( 'wpseo_schema_article', 'byline_feed_yoast_article', 10, 2 );

function byline_feed_yoast_article( array $data, $context ): array {
    $post    = get_post( $context->id );
    $authors = byline_feed_get_authors( $post );

    if ( empty( $authors ) ) {
        return $data;
    }

    // Replace Yoast's single @id reference with a full Person array.
    // Using inline objects rather than @id refs so the data is self-contained
    // when NLWeb ingests the schemamap endpoint — agents should not need to
    // resolve graph references to get author identity.
    $data['author'] = array_map(
        fn( $author ) => byline_feed_build_person_object( $author ),
        $authors
    );

    // Add perspective as additionalProperty if WP-03 meta is set.
    $perspective = get_post_meta( $post->ID, '_byline_perspective', true );
    if ( $perspective ) {
        $data['additionalProperty']   = $data['additionalProperty'] ?? [];
        $data['additionalProperty'][] = [
            '@type' => 'PropertyValue',
            'name'  => 'bylinePerspective',
            'value' => $perspective, // 'reporting' | 'opinion' | 'analysis' | 'sponsored'
        ];
    }

    return apply_filters( 'byline_feed_yoast_article_data', $data, $post, $authors );
}
```

### A2 — Enrich Person nodes already in the Yoast graph

Hook: `wpseo_schema_person_data` (filters each Person node; receives user_id).

```php
add_filter( 'wpseo_schema_person_data', 'byline_feed_yoast_person', 10, 3 );

function byline_feed_yoast_person( array $data, int $user_id, $context ): array {
    $post    = get_post( $context->id );
    $authors = byline_feed_get_authors( $post );
    $matched = array_filter( $authors, fn( $a ) => (int) $a->user_id === $user_id );

    if ( empty( $matched ) ) {
        return $data;
    }

    $author = array_values( $matched )[0];

    // Merge profiles and fediverse handle into sameAs.
    // Yoast may already carry sameAs from its social profile settings — merge, never overwrite.
    $extra = [];
    foreach ( $author->profiles as $profile ) {
        if ( ! empty( $profile['href'] ) ) {
            $extra[] = $profile['href'];
        }
    }
    // Fediverse handle: output canonical profile URL if resolvable, else omit.
    // A bare @user@instance handle is not a URI and should not go in sameAs directly.
    if ( ! empty( $author->fediverse ) ) {
        $fediverse_url = byline_feed_fediverse_profile_url( $author->fediverse );
        if ( $fediverse_url ) {
            $extra[] = $fediverse_url;
        }
    }

    if ( ! empty( $extra ) ) {
        $existing        = $data['sameAs'] ?? [];
        $data['sameAs']  = array_values( array_unique( array_merge( $existing, $extra ) ) );
    }

    // Add contributor role as additionalProperty.
    if ( ! empty( $author->role ) ) {
        $data['additionalProperty']   = $data['additionalProperty'] ?? [];
        $data['additionalProperty'][] = [
            '@type' => 'PropertyValue',
            'name'  => 'bylineRole',
            'value' => $author->role,
            // Values: 'creator' | 'editor' | 'guest' | 'staff' | 'contributor' | 'bot'
        ];
    }

    // Add ai_consent as additionalProperty if WP-06 signal is set.
    if ( ! empty( $author->ai_consent ) ) {
        $data['additionalProperty'][] = [
            '@type' => 'PropertyValue',
            'name'  => 'aiTrainingConsent',
            'value' => $author->ai_consent, // 'allow' | 'deny'
        ];
    }

    return apply_filters( 'byline_feed_yoast_person_data', $data, $author, $context );
}
```

### A3 — Add ai_consent graph piece (when WP-06 active)

When the site-level or per-post ai_consent signal is set, add a new graph node. This is the
schema.org equivalent of TDM headers — machine-readable consent that NLWeb agents can
inspect at the schemamap level without crawling individual pages.

```php
add_filter( 'wpseo_schema_graph_pieces', 'byline_feed_yoast_graph_pieces', 10, 2 );

function byline_feed_yoast_graph_pieces( array $pieces, $context ): array {
    if ( byline_feed_ai_consent_enabled() ) {
        $pieces[] = new Byline_Feed_AI_Consent_Schema_Piece( $context );
    }
    return $pieces;
}
```

`Byline_Feed_AI_Consent_Schema_Piece` extends `Abstract_Schema_Piece` and emits:

```json
{
  "@type": "CreativeWork",
  "@id": "https://example.com/#ai-consent-declaration",
  "name": "AI Training Consent Declaration",
  "conditionsOfAccess": "no-ai-training"
}
```

The `conditionsOfAccess` value is provisional pending Schema.org or TDM vocabulary
stabilization. Apply `byline_feed_schema_consent_node` filter for downstream customization.

## Mode B: Rank Math integration

Detection:

```php
function byline_feed_schema_rankmath_active(): bool {
    return class_exists( 'RankMath' )
        && class_exists( 'RankMath\Schema\Entities' );
}
```

Rank Math exposes a `rank_math/json_ld` filter over the full assembled JSON-LD array.
The integration pattern is less granular than Yoast's graph-piece system but achieves
equivalent enrichment:

```php
add_filter( 'rank_math/json_ld', 'byline_feed_rankmath_json_ld', 10, 2 );

function byline_feed_rankmath_json_ld( array $data, $jsonld ): array {
    $post    = get_post();
    $authors = byline_feed_get_authors( $post );

    if ( empty( $authors ) || ! is_singular() ) {
        return $data;
    }

    foreach ( $data as $key => $node ) {
        if ( isset( $node['@type'] ) && in_array( 'Article', (array) $node['@type'], true ) ) {
            $data[ $key ]['author'] = array_map(
                fn( $author ) => byline_feed_build_person_object( $author ),
                $authors
            );

            $perspective = get_post_meta( $post->ID, '_byline_perspective', true );
            if ( $perspective ) {
                $data[ $key ]['additionalProperty'][] = [
                    '@type' => 'PropertyValue',
                    'name'  => 'bylinePerspective',
                    'value' => $perspective,
                ];
            }
        }
    }

    return apply_filters( 'byline_feed_rankmath_json_ld', $data, $post, $authors );
}
```

Rank Math does not have per-node Person filters analogous to Yoast's `wpseo_schema_person_data`.
In Mode B, Person enrichment (sameAs merge, role, ai_consent) is applied inline within
`byline_feed_build_person_object()` rather than as a secondary filter pass.

## Shared builder: `byline_feed_build_person_object()`

Used by all three modes. Lives in `inc/schema.php`.

```php
function byline_feed_build_person_object( object $author ): array {
    $person = [ '@type' => 'Person' ];

    if ( ! empty( $author->display_name ) ) {
        $person['name'] = $author->display_name;
    }

    // URL: explicit author URL, then WP author archive as fallback.
    if ( ! empty( $author->url ) ) {
        $person['url'] = $author->url;
    } elseif ( $author->user_id ) {
        $person['url'] = get_author_posts_url( $author->user_id );
    }

    if ( ! empty( $author->description ) ) {
        $person['description'] = $author->description;
    }

    if ( ! empty( $author->avatar_url ) ) {
        $person['image'] = [
            '@type' => 'ImageObject',
            'url'   => $author->avatar_url,
        ];
    }

    // sameAs: profiles array + fediverse canonical URL.
    $same_as = array_column( $author->profiles ?? [], 'href' );
    if ( ! empty( $author->fediverse ) ) {
        $fediverse_url = byline_feed_fediverse_profile_url( $author->fediverse );
        if ( $fediverse_url ) {
            $same_as[] = $fediverse_url;
        }
    }
    if ( ! empty( $same_as ) ) {
        $person['sameAs'] = array_values( array_unique( $same_as ) );
    }

    // additionalProperty: role and ai_consent (omitted if empty).
    $additional = [];
    if ( ! empty( $author->role ) ) {
        $additional[] = [
            '@type' => 'PropertyValue',
            'name'  => 'bylineRole',
            'value' => $author->role,
        ];
    }
    if ( ! empty( $author->ai_consent ) ) {
        $additional[] = [
            '@type' => 'PropertyValue',
            'name'  => 'aiTrainingConsent',
            'value' => $author->ai_consent,
        ];
    }
    if ( ! empty( $additional ) ) {
        $person['additionalProperty'] = $additional;
    }

    return apply_filters( 'byline_feed_schema_person', $person, $author );
}
```

## Mode C: Standalone output (no SEO plugin)

When neither Yoast nor Rank Math is active, hook `wp_head` to output a self-contained
`<script type="application/ld+json">` block on singular post views.

```php
// Dispatch in inc/schema.php — called on plugins_loaded after adapter detection.
function byline_feed_schema_init(): void {
    if ( byline_feed_schema_yoast_active() ) {
        require_once __DIR__ . '/schema-yoast.php';
    } elseif ( byline_feed_schema_rankmath_active() ) {
        require_once __DIR__ . '/schema-rankmath.php';
    } else {
        require_once __DIR__ . '/schema-standalone.php';
    }
}
```

Standalone Article + Person output in `inc/schema-standalone.php`:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Post Title",
  "datePublished": "2026-03-07T12:00:00+00:00",
  "dateModified": "2026-03-07T14:00:00+00:00",
  "author": [
    {
      "@type": "Person",
      "name": "Jane Doe",
      "url": "https://example.com/author/jdoe",
      "description": "Staff writer covering technology.",
      "image": { "@type": "ImageObject", "url": "https://example.com/avatars/jdoe.jpg" },
      "sameAs": [ "https://mastodon.social/@jdoe", "https://github.com/jdoe" ],
      "additionalProperty": [
        { "@type": "PropertyValue", "name": "bylineRole",        "value": "creator" },
        { "@type": "PropertyValue", "name": "aiTrainingConsent", "value": "deny"    }
      ]
    }
  ],
  "publisher": {
    "@type": "Organization",
    "name": "Site Name",
    "url":  "https://example.com",
    "logo": { "@type": "ImageObject", "url": "https://example.com/logo.png" }
  }
}
```

`publisher` uses `get_bloginfo( 'name' )` and site icon URL. The `publisher.logo` field
is omitted if no site icon is set.

## Filter and hook API surface

| Filter | Signature | Purpose |
|--------|-----------|---------|
| `byline_feed_schema_mode` | `( string $mode )` | Override detected mode: `'yoast'`, `'rankmath'`, `'standalone'` |
| `byline_feed_schema_person` | `( array $person, object $author )` | Modify any Person object before output, all modes |
| `byline_feed_yoast_article_data` | `( array $data, WP_Post $post, array $authors )` | Modify enriched Article data before Yoast emits it |
| `byline_feed_yoast_person_data` | `( array $data, object $author, $context )` | Modify enriched Person data in Yoast graph pass |
| `byline_feed_rankmath_json_ld` | `( array $data, WP_Post $post, array $authors )` | Modify full Rank Math JSON-LD array after enrichment |
| `byline_feed_schema_consent_node` | `( array $node, $context )` | Modify the ai_consent graph piece |

## PHPUnit test coverage (`tests/phpunit/test-schema.php`)

### Mode detection
- Yoast active: `byline_feed_schema_yoast_active()` returns true, Rank Math returns false.
- Rank Math active: reverse.
- Neither: standalone mode dispatch fires.
- `byline_feed_schema_mode` filter overrides detection.

### Mode A — Yoast enrichment
- `wpseo_schema_article` filter: Article.author is replaced with ordered multi-author array.
- Single author post: Article.author is an array with one Person (not a bare @id string).
- Multi-author post: Article.author is an ordered array of Person objects.
- `perspective` meta set: Article.additionalProperty contains `bylinePerspective` PropertyValue.
- `perspective` meta absent: no `additionalProperty.bylinePerspective` in output.
- `wpseo_schema_person_data` filter: sameAs merged (not overwritten) from profiles array.
- `wpseo_schema_person_data` filter: fediverse handle resolved to canonical URL in sameAs.
- `wpseo_schema_person_data` filter: bylineRole PropertyValue present when role is set.
- `wpseo_schema_person_data` filter: aiTrainingConsent PropertyValue present when ai_consent set.
- `wpseo_schema_person_data` filter: no additionalProperty when role and ai_consent both empty.
- Person not in normalized authors (e.g. site owner): filter returns data unchanged.
- `wpseo_schema_graph_pieces`: ai_consent piece added when WP-06 signal active.
- `wpseo_schema_graph_pieces`: ai_consent piece absent when signal not set.

### Mode B — Rank Math enrichment
- `rank_math/json_ld` filter: Article node author replaced with multi-author array.
- `rank_math/json_ld` filter: perspective PropertyValue added when meta set.
- Non-singular page: filter returns data unchanged.

### Mode C — Standalone
- Singular post: `<script type="application/ld+json">` present in `wp_head` output.
- Non-singular page: no output.
- Single author: Article.author is array with one Person.
- Multi-author: Article.author is ordered array.
- Person fields: name, url, description, image, sameAs all populated from normalized object.
- Guest author with no URL: `url` field omitted from Person.
- `sameAs` omitted when profiles empty and no fediverse handle.
- `publisher.logo` omitted when no site icon.

### Shared
- `byline_feed_schema_person` filter fires for every Person object in all three modes.
- Profiles with empty `href` values are not included in sameAs.
- Bare fediverse handle that cannot be resolved to a URL is not included in sameAs.

## Notes

- The core architectural shift from the previous version of this work package: Yoast detection
  no longer triggers a disable path. It triggers an enrichment path. When Yoast's schemamap
  is the NLWeb endpoint that AI agents query, our most valuable action is to enrich that graph,
  not stand aside.
- `sameAs` handling: Yoast's social profile settings are the user-configured source; our
  profiles array is the live plugin data source. Merging both produces the most complete
  identity record. We never overwrite Yoast's existing values.
- The `bylineRole` and `bylinePerspective` property names are provisional and not Schema.org
  vocabulary. They are namespaced informally via `additionalProperty` to avoid claiming
  canonical Schema.org meaning. If Schema.org or a successor spec formalizes these concepts,
  the property names should be updated.
- The fediverse handle in `sameAs` requires a canonical HTTPS URL, not a bare `@user@instance`
  string. `byline_feed_fediverse_profile_url()` must resolve `@user@instance` to
  `https://instance/users/user` via WebFinger or a simple string transform. A failed
  resolution should be logged (non-fatal) and the field omitted rather than emitting an
  invalid URI.
- The `publisher` Organization in standalone mode should use `get_bloginfo( 'name' )` and
  site icon. A future enhancement could allow site-level Organization configuration matching
  Byline's `byline:org` element.
- The `sameAs` array is the JSON-LD equivalent of Byline's `byline:profile` elements. Same
  data source, different output format. Single-source-of-truth principle applied.
- This work package can be developed in parallel with WP-02 and WP-04 after WP-01 is complete.
  WP-03 and WP-06 enrichments are progressive — they add fields when available, they are not
  prerequisites for schema output to function.
