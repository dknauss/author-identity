WORK PACKAGE 04: `fediverse:creator` meta tag output

Goal:
- Output `<meta name="fediverse:creator">` tags in HTML page heads so that links shared on Mastodon (and other fediverse platforms) display author bylines.

Primary targets:
- `inc/fediverse.php`
- `tests/phpunit/test-fediverse.php`

Depends on:
- WP-01 (adapter interface and normalized author data).

Problem statement:
- Mastodon's author attribution feature (launched July 2024) reads `<meta name="fediverse:creator" content="@user@instance" />` from shared links to display clickable author bylines. WordPress sites with multi-author content currently have no automated way to output these tags. The feature was designed for journalism use cases and works with any fediverse account.

Planned changes:

1. Register user meta field for fediverse handle:
   ```php
   register_meta( 'user', 'byline_feed_fediverse', [
       'type'          => 'string',
       'single'        => true,
       'show_in_rest'  => true,
       'auth_callback' => function() {
           return current_user_can( 'edit_users' ) || get_current_user_id() === $user_id;
       },
   ] );
   ```

2. Add fediverse handle field to user profile edit screen:
   - Hook `show_user_profile` and `edit_user_profile` to render input field.
   - Hook `personal_options_update` and `edit_user_profile_update` to save.
   - Label: "Fediverse Handle" with description: "Your Mastodon or fediverse account (e.g. @you@mastodon.social). Used for author attribution when links are shared on the fediverse."
   - Validate format: must match `@?[\w.-]+@[\w.-]+` or be empty.

3. Output meta tags in `wp_head` (`inc/fediverse.php`):
   - Hook `wp_head` with a callback that runs only on singular post views.
   - Call `byline_feed_get_authors( $post )` for the current post.
   - For each author with a non-empty `fediverse` field, output:
     ```html
     <meta name="fediverse:creator" content="@user@instance" />
     ```
   - Normalize handle to include leading `@` if missing.
   - Apply `byline_feed_fediverse_handle` filter per author.

4. Adapter integration:
   - Each adapter should populate the `fediverse` field on the normalized author object from:
     - User meta `byline_feed_fediverse` for WP users.
     - Empty string for guest authors without a linked WP user (no fediverse handle available).
   - The core adapter and any adapter resolving to a `WP_User` can read this meta directly.
   - The CAP and PPA adapters should check if the author maps to a WP user and read the meta from that user.

5. Write PHPUnit tests (`tests/phpunit/test-fediverse.php`):
   - Single author with fediverse handle: meta tag present in `wp_head` output.
   - Single author without handle: no meta tag output.
   - Multiple authors, some with handles: correct number of meta tags.
   - Handle normalization: `user@instance` becomes `@user@instance`.
   - Filter overrides handle.
   - Non-singular pages (archives, home): no meta tag output.
   - Guest authors without linked WP users: no meta tag (graceful absence).

Validation:
- Singular post pages include `<meta name="fediverse:creator">` tags for authors with fediverse handles.
- Tags are absent for authors without handles and on non-singular pages.
- User profile edit screen shows fediverse handle field.
- Handle format is validated on save.
- Filter works for programmatic override.
- PHPUnit suite green.
- PHPCS clean.

Notes:
- Mastodon currently displays only the first `fediverse:creator` tag when multiple are present. Output all of them anyway — Mastodon has stated intent to support multiple authors and has introduced an `authors` array in its REST API (PR #30846). Being ready for multi-author display when it ships is better than having to update later.
- The `fediverse:creator` tag requires the author to have added the publishing domain to their "Author Attribution" list in their Mastodon profile settings. This is a Mastodon-side verification step that the plugin cannot control. Document this in the plugin's readme.
- This work package is independent of WP-02 (feed output) and WP-03 (perspective). It can be developed in parallel after WP-01.
