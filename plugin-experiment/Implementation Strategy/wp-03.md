WORK PACKAGE 03: Perspective meta field and editor UI

Goal:
- Give editors a way to set the Byline `perspective` value per post, and output it in feeds. This is the MVP's editorial feature — the element that distinguishes analysis from press releases in a reader.

Primary targets:
- `inc/perspective.php`
- `src/perspective-panel.tsx`
- `tests/phpunit/test-perspective.php`

Depends on:
- WP-01 (namespace and hook infrastructure).
- WP-02 (feed output consumes the perspective value).

Problem statement:
- No WordPress data field natively represents content perspective (opinion vs. reporting vs. tutorial vs. satire). The Byline spec defines a `perspective` element with an enumerated vocabulary. This needs a storage mechanism, an editorial UI, and a filter for programmatic computation.

Planned changes:

1. Register post meta (`inc/perspective.php`):
   ```php
   register_post_meta( '', '_byline_perspective', [
       'type'          => 'string',
       'single'        => true,
       'show_in_rest'  => true,
       'auth_callback' => function() {
           return current_user_can( 'edit_posts' );
       },
   ] );
   ```
   Register for all post types (empty string first arg) so it's available on any post type that supports author.

2. Public API function (`inc/perspective.php`):
   ```php
   function byline_feed_get_perspective( \WP_Post $post ) : string {
       $perspective = get_post_meta( $post->ID, '_byline_perspective', true );
       $perspective = apply_filters( 'byline_feed_perspective', $perspective, $post );

       // Validate against allowed values
       $allowed = [
           'personal', 'reporting', 'analysis', 'official', 'sponsored',
           'satire', 'review', 'announcement', 'tutorial', 'curation',
           'fiction', 'interview',
       ];

       if ( ! in_array( $perspective, $allowed, true ) ) {
           return '';
       }

       return $perspective;
   }
   ```

3. Block editor sidebar panel (`src/perspective-panel.tsx`):
   - Register via `registerPlugin` using `PluginDocumentSettingPanel`.
   - Render a `<SelectControl>` with options:
     - (empty) — "No perspective set"
     - `personal` — "Personal / Opinion"
     - `reporting` — "News Reporting"
     - `analysis` — "Analysis / Commentary"
     - `official` — "Official Statement"
     - `sponsored` — "Sponsored Content"
     - `satire` — "Satire / Humor"
     - `review` — "Review / Critique"
     - `announcement` — "Announcement / Release Notes"
     - `tutorial` — "Tutorial / How-to"
     - `curation` — "Curated Links"
     - `fiction` — "Creative Fiction"
     - `interview` — "Interview / Q&A"
   - Read/write via `useEntityProp( 'postType', postType, 'meta' )`.
   - Panel title: "Content Perspective" with a brief description: "How should readers interpret this content? This appears in your feed's Byline metadata."

4. Classic editor support (`inc/perspective.php`):
   - Register a meta box with the same select dropdown for sites not using the block editor.
   - Save via `save_post` hook with nonce verification.

5. Filter documentation:
   - The `byline_feed_perspective` filter receives `( $perspective, $post )` and can return any allowed value.
   - Example in inline docblock:
     ```php
     // Compute perspective from categories
     add_filter( 'byline_feed_perspective', function( $perspective, $post ) {
         if ( $perspective ) return $perspective; // Don't override explicit setting
         if ( has_category( 'opinion', $post ) ) return 'personal';
         if ( has_category( 'news', $post ) ) return 'reporting';
         return $perspective;
     }, 10, 2 );
     ```

6. Write PHPUnit tests (`tests/phpunit/test-perspective.php`):
   - Setting meta returns correct value from `byline_feed_get_perspective()`.
   - Invalid values are rejected (returns empty string).
   - Filter overrides meta value.
   - Filter does not override when meta is explicitly set (per the example pattern).
   - Empty perspective results in no `<byline:perspective>` element in feed output.

Validation:
- Block editor shows "Content Perspective" panel in document sidebar.
- Selecting a perspective and saving the post persists the value in post meta.
- RSS2 and Atom feed output includes `<byline:perspective>` when set, omits when empty.
- The `byline_feed_perspective` filter works for programmatic computation.
- Invalid values are silently dropped (no output, no error).
- PHPUnit suite green.
- PHPCS clean.
- JSX/TSX passes eslint.

Notes:
- The perspective vocabulary matches the Byline spec v0.1.0 enumeration exactly. If the spec adds values, the allowed list and select options should be updated.
- The filter pattern (check if meta is set before computing) should be documented prominently so theme developers don't accidentally override editorial choices.
